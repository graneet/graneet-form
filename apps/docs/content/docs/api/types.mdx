## Path Types

### FieldPath

Union of all valid field names for a form model `T` — every top-level key **plus** every nested object path
(objects only, no array indices). Used to constrain `name` on `Field`, `useFieldsWatch`, `useValidations`, etc.

```tsx
interface ProfileForm {
  email: string;
  user: { name: string; address: { city: string } };
}

type Paths = FieldPath<ProfileForm>;
// 'email' | 'user' | 'user.name' | 'user.address' | 'user.address.city'
```

`FieldPath<T>` is a superset of `keyof T`, so existing flat field names keep type-checking. See
[Nested Object Paths](/docs/form/nested-paths).

### FieldPathValue

Resolves the value type located at a given path within `T`:

```tsx
type City = FieldPathValue<ProfileForm, 'user.address.city'>; // string
type Address = FieldPathValue<ProfileForm, 'user.address'>; // { city: string }
```

## Usage Examples

### Defining Form Types

```tsx
// Define your form data structure
interface UserRegistrationForm {
  firstName: string;
  lastName: string;
  email: string;
  age: number;
  preferences: {
    newsletter: boolean;
    marketing: boolean;
  };
}

// Use with useForm
const form = useForm<UserRegistrationForm>({
  defaultValues: {
    firstName: '',
    lastName: '',
    email: '',
    age: 18,
    preferences: {
      newsletter: false,
      marketing: false,
    },
  },
});
```

### Working with Validation

```tsx
// Custom validation function
const validateEmail = (value: string): boolean => {
  return /^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(value);
};

// Use in Field component
<Field<UserRegistrationForm, 'email'>
  name="email"
  render={(props, state) => (
    <div>
      <input {...props} type="email" />
      {state.validationStatus.status === 'invalid' && <span>{state.validationStatus.message}</span>}
    </div>
  )}
>
  <Rule validationFn={validateEmail} message="Please enter a valid email" />
</Field>;
```

### Type-safe Field Access

```tsx
// Get values with proper typing
const formValues = form.getFormValues();
// formValues is typed as Partial<UserRegistrationForm> (nested objects reconstructed)

// Set values with type safety
form.setFormValues({
  firstName: 'John', // ✅ Valid
  email: 'john@example.com', // ✅ Valid
  // invalidField: 'value' // ❌ TypeScript error
});

// Nested objects can be updated partially (DeepPartial)
form.setFormValues({
  preferences: { newsletter: true },
});
```

### Custom Hook with Types

```tsx
function useUserForm() {
  const form = useForm<UserRegistrationForm>({
    defaultValues: {
      firstName: '',
      lastName: '',
      email: '',
      age: 18,
      preferences: {
        newsletter: false,
        marketing: false,
      },
    },
  });

  const handleSubmit = form.handleSubmit(async (data) => {
    // data is fully typed as UserRegistrationForm
    await submitUserRegistration(data);
  });

  return { form, handleSubmit };
}
```

## Generic Type Constraints

Most Graneet Form types are generic and constrained to ensure type safety:

```tsx
// T extends FieldValues ensures form data is a valid object
useForm<T extends FieldValues>()

// K extends FieldPath<T> ensures field names are valid keys or nested paths
Field<T extends FieldValues, K extends FieldPath<T>>

// Validation functions receive properly typed values
Rule<T extends FieldValues, K extends FieldPath<T>>
```
